## 🚀 Choose Your Integration Path

ELENX is designed to be platform-agnostic. Depending on your use case, you can integrate our security layer in four distinct ways:

### 1. [The Developer SDK (Core Middleware)](./SDK_GUIDE.md)
*   **Best for:** AI Engineering teams building custom autonomous workflows.
*   **Form Factor:** NPM/PyPI Package (`@elenx/core`).
*   **Goal:** Native, high-performance semantic scrubbing inside your application logic.

### 2. [The Secure Tool API (REST Endpoint)](./API_GUIDE.md)
*   **Best for:** Custom GPTs, ChatGPT, and serverless agent frameworks.
*   **Form Factor:** REST API (/v1/scrub, /v1/verify).
*   **Goal:** Zero-config security middleware accessible via simple HTTP requests.

### 3. [The MCP Plugin (Claude & Cursor)](./MCP_GUIDE.md)
*   **Best for:** Individuals and small teams using frontier agents for daily work.
*   **Form Factor:** Model Context Protocol (MCP) Server.
*   **Goal:** Plug-and-play governance for Claude Desktop and AI IDEs.

### 4. [The Enterprise Extension (Browser Firewall)](./EXTENSION_GUIDE.md)
*   **Best for:** Corporate-wide deployment and CISO-level oversight.
*   **Form Factor:** Chrome/Edge Web Extension.
*   **Goal:** Real-time interception and governance of all agent activity on employee machines.
